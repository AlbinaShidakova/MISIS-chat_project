В этом файле описаны команды для общения между клинтом и сервером.

Все команды от клиента к серверу должны поступать в формате json.
у каждой команды может быть свой список атрибутов (См. пример).
Пример команды:
    {'cmd':'VK_Autorization', 'username' : "user1", 'password':"pass1"}

Поддерживаемые команды:
    VK_Autorization - авторизация пользователя через ВК:
        Аттрибуты:
            username - Имя пользователя в ВК
            password - пароль в ВК.
        Ответ:
            {"Error":"0"/"1", "Answer" : "Incorrect username or password"/"Autorization OK")

    [Need to be implemented]Login - авторизация в чате:
        Аттрибуты:
        Ответ:

    [Need to be implemented]Registration - регистрация в чате (заполнение анкеты):
        Аттрибуты:
        Ответ:

    get_msg - получить сообщение из бд:
    Аттрибуты:
            N - Кол-во сообщений (?)
        Ответ:
            {"Error":"0"/"1", "msg" : {...})

    send_msg - отправить сообщение
            text - Текст сообщения
            id_dialog - ID диалога, в перспективе - имя диалога
            id_user   - ID пользователя, в перспективе  - удаление аттрибута

    create_new_user - создать нового пользователя (регистрация на сайте)
            first_name  - Имя пользователя
            last_name   - Фамилия пользователя
            login       - уникальный(!) логин пользователя
            pas         - парольпользователя
            D_birth     - Дата рождения
            age         - Возраст, в перспективе - вычисляется автоматически
            sex         - Пол
            city        - Город

    create_new_dialog - создать новый диалог
            Name    - имя диалога
            create_date - Дата создания
            capacity - Вместимость, в перспективе автоматическое вычисление

    get_users - получить список пользователей с контактной информацией
        Поля:
            Name - Имя пользователя
            D_birth - Дата рождения
            age - Возраст, в перспективе - вычисляется автоматически
            sex - Пол
            city - Город

    get_dialogs - получить список диалогов,
        Поля:
            Name    - имя диалога
            create_date - Дата создания
            capacity - Вместимость, в перспективе автоматическое вычисление
